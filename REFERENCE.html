Forms

  • Advantage to using a field label: first, the label becomes clickable to focus the input, and second it helps screen readers.
  • Automatically add nice form spacing by wrapping (label+input) in a div with class='form-group'
  • Checkboxes are handled differently.

    Or like this:
  • Using class='form-control' makes an input use 100% width, plus other small tweaks. Use on select field and text-area too.
  • This should be centered or on right:
  • Use class='radio-inline' on the label for horiz radio buttons, but must also remove the wrapper div with class='radio'
  • for wide screens say md lg, use <form class='form-inline'
  • Horizontal forms. add <form class='form-horizontal'. Then add to the form-groups labels class='col-sm-2' and class='col-sm-2' for the input. In other words, the form-group acts as a .row and the form acts as a .container. And finally, the label needs an extra class called 'control-label'
    This is a very good format for desktops, then stack the label/input for mobile.

  • Reminder: Use placeholders
  • Reminder: Linking radio buttons is done by using the same name in each

Validation classes

has-warning, has-error, has-success
On the label component add a class='control-label' in order for it to reflect the status colors.
Adding validation feedback icons. This requires a text inputs with class='form-control'. First add a class='has-feedback' 

<div class='form-group has-success has-feedback'><label for="idInput2" class="control-label"></label>
<input id='idInput2' type="text" class="form-control">
<span class="glyphicon glyphicon-ok form-control-feedback"></span>
The form-control-feedback will add a checkmark on the right when the field is valid. Use aria-hidden="true" to ensure this icon will not confuse screen readers. Instead add something like this for screen readers only: <span id='nameStatus' class='sr-only'>Success</span> Finally, link this with the input field by adding this to the input: aria-describedby='nameStatus'
